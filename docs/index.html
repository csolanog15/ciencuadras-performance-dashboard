Recomendaciones Tecnicas

P0 Critico Cargar fotos del inmueble una por una Mejora: LCP (de 14s a 2.5s)

Las fotos del inmueble tardan 14 segundos en aparecer porque se descargan todas juntas. Hay que cargar solo la primera foto rapido y las demas conforme el usuario las necesite.

  1. En el componente de galeria (apps/detail/), activar lazy loading en Swiper con la opcion loadPrevNext.
  2. Marcar la primera imagen con fetchpriority="high" y loading="eager". Las demas con loading="lazy".
  3. Configurar el CDN (CloudFront) para servir imagenes en formato WebP automaticamente.
  4. Agregar width y height a todas las etiquetas img del carrusel.
P1 Alto Quitar codigo JavaScript que no se usa al inicio Mejora: TBT (de 720ms a 200ms)

La pagina se congela 720ms porque descarga librerias que no necesita al inicio (mapas, encriptacion). Hay que cargarlas solo cuando el usuario las necesite.

  1. Quitar @angular/google-maps y crypto-js del archivo webpack-shared-versions.js (linea 62).
  2. Cargar Google Maps solo cuando el usuario abre la seccion de mapa (lazy load por ruta).
  3. Diferir la carga de GTM, Datadog y chat widgets al evento onLoad (3 segundos despues).
  4. Verificar que criticalPenthouse.js esta generando CSS critico correctamente en el build de produccion.
P2 Medio Evitar que los elementos se muevan al cargar Mejora: CLS (de 0.106 a menos de 0.1)

En celular, las imagenes y los microfrontends aparecen sin espacio reservado, causando que todo se mueva. Hay que reservar el espacio antes de que cargue el contenido.

  1. Agregar width y height a todas las imagenes + CSS aspect-ratio: 4/3.
  2. Reservar min-height en el shell antes de montar cada microfrontend.
  3. Agregar font-display: swap a las tipografias para evitar parpadeo de texto.
  4. Configurar cache de fuentes en Nginx (default.conf) con max-age 1 anio.
P3 Mejora Acelerar la primera carga del Home Mejora: LCP Home (de 9s a 4s)

El home tarda 9 segundos en mostrar la imagen principal porque descarga demasiado codigo antes de pintarla. Hay que priorizar lo que el usuario ve primero.

  1. Agregar link preload para la imagen hero en el index.html del shell.
  2. Verificar que criticalPenthouse.js genera CSS inline para el above-the-fold.
  3. Agregar preconnect al CDN de imagenes (multimedia.dvisor.io).
  4. Activar compression gzip/brotli en Nginx para JS y CSS.
