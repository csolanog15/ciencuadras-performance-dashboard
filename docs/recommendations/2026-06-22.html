P0 - LCP Mobile en Detalles (13-15s → target 2.5s)

Causa raiz identificada en codigo

El carrusel de imagenes usa ngx-owl-carousel-o (v14.0.2) que NO soporta native lazy loading ni fetchpriority. Owl Carousel carga TODAS las imagenes del slide al inicializar. Existe ng-lazyload-image (v9.0.0) pero no se aplica al carrusel de detalle.

Acciones especificas

  1. Migrar de ngx-owl-carousel-o a ngx-splide (ya instalado v0.0.11). Splide soporta lazy loading nativo.
    - Buscar componente con <owl-carousel-o> en detalle de inmueble
    - Reemplazar por <splide> con config lazyLoad: 'nearby'
  2. Primera imagen con eager + fetchpriority
    <img [src]="images[0].url" loading="eager" fetchpriority="high" width="800" height="600">
<img *ngFor="let img of images.slice(1)" [lazyImage]="img.url" loading="lazy">
  3. Pre-render con Scully incluyendo <link rel="preload"> para imagen principal
  4. WebP via CloudFront - Repo ciencuadras-imagehandler-lambda maneja imagenes. Agregar content negotiation.

Impacto: LCP -8s (de 14s a ~6s).

P1 - TBT: JavaScript Bloqueante (280-2210ms → target 200ms)

Causa raiz identificada en codigo

Dependencias pesadas en bundle principal (package.json):

  • aws-sdk v2.804.0 = 250KB+ (tienen v3 client instalado pero no migrado)
  • firebase v9.1.0 = 100KB+ sin tree-shaking modular
  • @angular/google-maps + @googlemaps/js-api-loader = 200KB cargado sincrono
  • @ngrx/store-devtools cargado en produccion
  • @datadog/browser-rum inicializado en app.module.ts (bloquea render)

Acciones especificas

  1. Eliminar aws-sdk v2 - Ya tienen @aws-sdk/client-cognito-identity v3. Ahorro: ~250KB.
  2. Lazy load Google Maps - Mover a feature module con loadChildren. Ahorro: ~200KB.
  3. Remover store-devtools en produccion:
    ...(environment.production ? [] : [StoreDevtoolsModule.instrument()])
  4. Defer Datadog RUM:
    setTimeout(() => datadogRum.init({...}), 3000);
  5. Defer GTM (angular-google-tag-manager v1.6.0) - Cargar post-load.

Impacto: TBT -60% (de 800ms a ~320ms).

P2 - CLS Mobile > 0.1

Causa raiz

  • ngx-owl-carousel-o no reserva espacio antes de cargar imagenes
  • cc-components-lib (v0.0.611) no define aspect-ratios en cards
  • Web fonts sin font-display: swap

Acciones

  1. Agregar width/height a todos los <img> + aspect-ratio: 4/3
  2. CSS: .property-gallery { min-height: 400px; aspect-ratio: 16/9; }
  3. font-display: swap en styles.scss
  4. Skeleton screens para cards de resultados

Impacto: CLS de 0.107 a 0.03.

P3 - LCP Home Mobile (9s → target 2.5s)

Causa raiz

Scully pre-renderiza pero FCP de 6.5s indica hidratacion lenta. App.module.ts carga todo eagerly.

Acciones

  1. Verificar pre-render de ruta / en Scully
  2. <link rel="preload" as="image"> para hero
  3. <link rel="preconnect" href="https://multimedia.dvisor.io">
  4. Verificar "optimization": true en angular.json produccion

Impacto: LCP -5s (de 9s a ~4s).
