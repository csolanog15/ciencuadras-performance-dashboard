P0 — Critico Optimizar galeria de imagenes en detalle de inmueble

Las fotos del inmueble tardan mas de 13 segundos en aparecer porque se descargan todas a la vez en lugar de una por una.

Metrica:LCP Mobile (Largest Contentful Paint)
Actual:13.6s promedio en detalles
Objetivo:≤ 2.5s
Repo:ciencuadras-monorepo-mf → ciencuadras/apps/detail/
Causa:El microfrontend detail usa swiper (shared via Module Federation) pero las imagenes no tienen loading="lazy" ni fetchpriority. Todas se descargan al montar el componente.

Acciones concretas:

  1. Lazy load en Swiper: En el componente de galeria dentro de apps/detail/, configurar Swiper con lazy: { loadPrevNext: true, loadPrevNextAmount: 1 }. Solo la primera imagen se carga inmediatamente.
    Archivo: apps/detail/src/app/components/gallery/
  2. Primera imagen con prioridad maxima: Agregar fetchpriority="high" y loading="eager" al primer slide. El resto con loading="lazy".
    Impacto: LCP baja de 13s a ~5s solo con esto.
  3. Formato WebP con fallback: Configurar el CDN (CloudFront + ciencuadras-imagehandler-lambda) para servir WebP automaticamente via header Accept.
    Impacto: -40% peso de imagenes sin cambio visual.
  4. Dimensiones fijas en img: Agregar width="800" height="600" a todas las <img> del carrusel para evitar layout shift mientras carga.
P1 — Alto Reducir JavaScript bloqueante del shell y shared deps

La pagina se congela al cargar porque el navegador descarga codigo JavaScript que no necesita en ese momento.

Metrica:TBT (Total Blocking Time)
Actual:720ms mobile / 504ms desktop (hasta 2.2s en detalle desktop)
Objetivo:≤ 200ms
Repo:ciencuadras-monorepo-mf → ciencuadras/apps/shell/ y webpack-shared-versions.js
Causa:Module Federation comparte @angular/google-maps, swiper, crypto-js y @angular-slider/ngx-slider como singletons cargados al inicio — aunque no todas las rutas los necesitan.

Acciones concretas:

  1. Remover @angular/google-maps de shared deps: En webpack-shared-versions.js, quitar Google Maps del array standardAngularDeps. Cada MF que lo necesite lo carga en su propio bundle lazy.
    Archivo: ciencuadras/webpack-shared-versions.js linea ~62
  2. Remover crypto-js de shared: Solo lo usa auth. Moverlo al bundle del MF que lo necesita.
    Ahorro: ~50KB del bundle inicial.
  3. Diferir third-party scripts: En apps/shell/, mover la inicializacion de GTM, Datadog RUM y chat widgets al evento onLoad o con setTimeout(3000).
  4. Revisar criticalPenthouse.js: El archivo existe pero puede no estar generando Critical CSS correctamente. Verificar que la tarea Gulp ejecuta y produce CSS inline.
    Archivo: ciencuadras/criticalPenthouse.js + gulpfile.js
P2 — Medio Estabilizar layout en mobile (eliminar saltos visuales)

Al cargar la pagina en celular, los elementos se mueven y generan una experiencia inestable que confunde al usuario.

Metrica:CLS (Cumulative Layout Shift)
Actual:0.106 mobile (supera umbral 0.1)
Objetivo:≤ 0.1
Repo:ciencuadras-monorepo-mf → apps/detail/ y apps/result/
Causa:Imagenes sin dimensiones definidas + tipografias web sin fallback + MFs que se montan desplazando contenido existente.

Acciones concretas:

  1. Aspect ratio en imagenes: En apps/detail/ y apps/result/, agregar width/height a todas las <img> + CSS aspect-ratio: 4/3; object-fit: cover;
  2. Placeholder del MF antes de montar: En el shell, reservar el espacio del microfrontend con min-height antes de que se cargue via Module Federation.
  3. font-display: swap en la carga de tipografias (verificar en apps/shell/ styles globales).
  4. Configurar Nginx (default.conf) con headers de cache para fuentes: Cache-Control: public, max-age=31536000.
P3 — Mejora Mejorar carga inicial del Home

La pagina principal tarda 9 segundos en mostrar contenido util porque descarga demasiado antes de pintar algo en pantalla.

Metrica:LCP Home Mobile
Actual:9.1s
Objetivo:≤ 2.5s
Repo:ciencuadras-monorepo-mf → apps/shell/ + ciencuadras-portal-frontend (Scully)
Causa:Hero image sin preload + Critical CSS (criticalPenthouse.js) aparentemente no aplicado + shell carga todos los shared deps antes de renderizar.

Acciones concretas:

  1. Preload hero image: En el index.html del shell agregar <link rel="preload" as="image" href="URL_HERO">.
  2. Verificar Critical CSS: Ejecutar criticalPenthouse.js y confirmar que genera CSS inline en el build de produccion. Si no, corregir rutas en gulpfile.js.
  3. Preconnect al CDN: <link rel="preconnect" href="https://multimedia.dvisor.io">
  4. Nginx compression: Verificar que default.conf tiene gzip on para JS/CSS y brotli si es posible.
